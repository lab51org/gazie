 Questo è un sistema che, tramite appositi file inseriti nell'e-commerce, permette di interfacciare GAzie ad un sito internet scambiando i dati da un database ad un altro. Lo scambio dei dati fra DataBase avviane tramite file xml trasferiti via FTP.
 Grazie alla sua semplicità tecnica questo sistema è adattabile ad ogni applicazione che utilizzi tabelle di database.
L'adattabilità fra i vari CMS e siti e-commerce è realizzata creando specifici file di interfaccia mentre il modulo di GAzie "Shop-synchronize" rimane sempre lo stesso.
 Il meccanismo di funzionamento è semplice; per ovviare al fatto che la maggior parte dei database non sono gestibili da programmi residenti all'esterno del dominio cui il database appartiene, si caricano i dati da trasferire tramite un file xml via FTP e poi si avvia ul programma, residente nel dominio del database da modificare, che si occuperà di aggiornare il data base.

*ATTIVAZIONE* 
Per attivare la sincronizzazione manuale basta semplicemente mantere attivo questo modulo e cliccare sui relativi pulsanti di sincronizzazione presenti alla pagina "Sincronizza"
Esiste, poi, anche una sincronizzazione automatica attivale dalla pagina impostazioni, raggiungibile cliccando su "Impostazioni" sempre sulla pagina "Sincronizza"; è possibile impostare anche l'intervallo di tempo fra una sincronizzazione e l'altra. 
Poi si consiglia di attivare anche il relativo widget in home page. 

*PRIMA DELL'USO*,
Nella pagina "Sincronizza", cliccando sul pulsante "Impostazioni", si dovranno inserire i dati di accesso FTP all'host del sito internet e il percorso per l'upload ftp, su Website root directory si dovrà inserire il percorso della cartella su cui risiederanno le interfaccie (ad esempio: https://www.tuosito.it/syncro/ ). 
Per evitare intrusioni indesiderate, l'interfaccia di GAzie, al momento di connettersi con l'e-commerce, passa, tramite l'url, una password. Tale 
password è, per comodità, la stessa che GAzie ha per l' FTP e che viene memorizzata su configurazione avanzata azienda. L'interfaccia del sito dovrà, quindi,fare un controllo sulla password che ha ricevuto da GAzie e, se confermata, si avvierà.

 *DOWNLOAD ORDINI*
Il file di interfaccia che si inserirà nel percorso impostato del sito web/e-commerce si dovrà chiamare: ordini-gazie.php.
 
*UPLOAD QUANTITà ARTICOLI (veloce)*
Il file di interfaccia che si inserirà nel percorso impostato del sito web/e-commerce si dovrà chiamare: articoli-gazie.php

*VERSIONE 2.0*
Con la versione 2.0 questo modulo si completa con una maggiore sincronizzazione riuscendo a gestire anche le immagini, i prezzi e le descrizioni estese degli articoli.
Inoltre la sincronizzazione è bidirezionale:
-si possono importare, creandoli se inesistenti, gli articoli dall'ecommerce a GAzie
-si possono aggiornare gli articoli da GAzie all'ecommerce nei prezzi, immagini e descrizioni estese.

Dall'e-commerce a GAzie (il file di interfaccia si dovrà chiamare: dwnlArticoli-gazie.php)


* FILE DI INTERFACCIA DA INSERIRE NEL SITO *
 I file di interfaccia da inserire nel sito devono essere creati in base alle esigenze e alle impostazioni dell'applicazione installata nel sito.
 
Nel sito di Antonio Germani sono disponibili i file di interfaccia per Joomla! e per WordPress così come è possibile richiedere ulteriori file di interfaccia specifici per altri e-commerce: https:www.programmisitiweb.lacasettabio.it
